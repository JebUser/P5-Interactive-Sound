# Automobile Data Sonifier & Visualizer

**Project for: Systems of Interaction - Mini Project 5**

## Overview

This project is an **interactive audio-visual data exploration system** that transforms automobile performance data into synchronized sound and graphics. By combining Processing for visualization with Pure Data for sound synthesis, the project creates an immersive way to explore relationships between vehicle characteristics and their perceptual equivalents through sound.

The system reads vehicle data from the Kaggle Automobile dataset and generates unique audio signatures for each car based on a mathematical formula (generated by AI) that combines multiple vehicle parameters into a single meaningful sound.

## Technical Architecture

### Data Flow

```
Automobile.csv (398 vehicles)
        ↓
Processing Application
├─ Load CSV data
├─ Visualize each vehicle
└─ Calculate sonification parameter
        ↓
        OSC/UDP Protocol (localhost:12001)
        ↓
Pure Data Patch
├─ Receive sonification parameter
├─ Generate audio waveform
└─ Output sound
```

### Communication Protocol

- **Protocol**: Open Sound Control (OSC) over UDP
- **Sender Port**: 8000 (Processing → Pure Data)
- **Messages**:
  - `/carData <60-90>` - Sonification parameter (integer)
  - `/carPlay <0|1>` - Play/Pause toggle

---

### Mathematical Formula

The core of this project is a **mathematical transformation** that converts multi-dimensional vehicle data into a single meaningful sound parameter:

```
rawValue = (MPG × 3) + (HP ÷ 5) - (Cylinders × 2) + (Weight ÷ 100) + Origin

Where:
- MPG: Miles per gallon (9-46)
- HP: Horsepower (46-230)
- Cylinders: Number of cylinders (3, 4, 6, 8)
- Weight: Vehicle weight in pounds (1613-5140)
- Origin: Country code (USA=1, Japan=2, Europe=3)

Final Step: Map rawValue to range 60-90
finalValue = map(rawValue, 37.33, 232.4, 60, 90)
```

### Step 1: Install Software

1. Download and install Processing from https://processing.org
2. Open Processing → Sketch → Import Library → Manage Libraries
3. Search for "oscP5" (by Andreas Schlegel) and install
4. Download Pure Data vanilla from https://puredata.info
5. Install Pure Data

### Step 2: Organize Files

Put everything in main folder

### Step 3: Run the System

1. **Start Pure Data first**:

   - Open `automobile_sonifier.pd`
   - Check console for: `netreceive: listening on port 12001`
   - Enable DSP: Check "Compute Audio" box or press Ctrl+/

2. **Start Processing second**:

   - Open `automobile_visualizer_v3.pde`
   - Click Run (▶) or press Ctrl+R
   - Visualization window appears

3. **Interact**:
   - **SPACEBAR**: Play/Pause (toggles sound on/off)
   - **LEFT ARROW**: Previous vehicle
   - **RIGHT ARROW**: Next vehicle

## How It Works

### Processing Side

1. **Data Loading**

   - Reads Automobile.csv with 398 vehicle records
   - Parses 9 data fields per vehicle

2. **Visualization**

   - MPG: Circle size and color (red=inefficient, green=efficient)
   - Horsepower: Bar height
   - Weight: Rectangle size
   - Cylinders: Number of circles
   - Origin: Color code (USA=red, Japan=green, Europe=blue)
   - Year and other data: Text display

3. **Sonification Calculation**

   - Applies the mathematical formula for each vehicle
   - Generates value between 60-90
   - Sends only when vehicle changes (no overflow)

4. **OSC Communication**
   - `/carData <value>` when vehicle changes
   - `/carPlay <1|0>` when SPACEBAR is pressed
   - Efficient, sparse messaging

### Pure Data Side

1. **Reception**

   - Listens on UDP port 12001
   - Receives two OSC message types

2. **Signal Processing**

   - `/carData`: Maps 60-90 range to frequency (200-1200 Hz)
   - `/carPlay`: Uses 0/1 value as amplitude gate

3. **Sound Generation**

   - Sine wave oscillator controlled by frequency
   - Amplitude multiplier acts as on/off switch
   - Smooth envelope to avoid audio artifacts

## Data Analysis

### Dataset: UCI Auto MPG

- **Records**: 398 automobiles
- **Time Period**: 1970-1982
- **Origins**: USA, Japan, Europe

### Variables Used

| Variable     | Range       | Purpose                |
| ------------ | ----------- | ---------------------- |
| MPG          | 9-46        | Efficiency coefficient |
| Horsepower   | 46-230      | Power coefficient      |
| Displacement | 68-455      | (informational)        |
| Weight       | 1613-5140   | Mass coefficient       |
| Acceleration | 8-24.6      | (informational)        |
| Cylinders    | 3,4,6,8     | Complexity coefficient |
| Model Year   | 70-82       | Timeline               |
| Origin       | 3 countries | Regional coefficient   |

### Sonification Range

The final sonification parameter is always **60-90** (integer):

- Lower values (60-70): Efficient, lightweight, powerful vehicles
- Middle values (70-75): Average vehicles
- Higher values (75-90): Heavy, simple engines

This range is then converted to audio frequency:

- 60 → ~200 Hz (deep bass)
- 90 → ~1200 Hz (high pitch)

## Technical Implementation Details

### Processing Code Highlights

```java
// Main sonification equation
float rawValue = (mpgValue * 3) + (hpValue / 5)
                 - (cylValue * 2) + (weightValue / 100)
                 + originNum;

// Map to 60-90 range
float mappedValue = map(rawValue, 37.33, 232.4, 60, 90);
int finalValue = round(mappedValue);

// Send only when car changes
if (currentCarIndex != lastSentCarIndex) {
  OscMessage msg = new OscMessage("/car/data");
  msg.add(finalValue);
  oscP5.send(msg, pureData);
  lastSentCarIndex = currentCarIndex;
}

// Send play/pause toggle
OscMessage msgPlay = new OscMessage("/car/play");
msgPlay.add(isPlaying ? 1 : 0);
oscP5.send(msgPlay, pureData);
```
